# globals.css Template

This is the structural template for the CSS token file written by `init-design-system`. The `token-expander` agent (Mode B) or the `bundle-validator` agent (Mode A) supplies the actual values. This file defines the shape and organization only.

**Target path:** `app/globals.css` or `src/app/globals.css` — detect from project structure.

## Rules for Filling This Template

1. **Never copy placeholder values** (`<value>`, `<dark-value>`) into an actual project — they must be replaced with real token values before the gate can pass.
2. **Omit optional sections** if not applicable (sidebar, optional semantic colors, optional font families). Do not leave placeholder lines for unused tokens.
3. **Motion tokens**: if the project uses Tailwind theme extensions for motion rather than CSS custom properties, remove the motion block from this file and document the decision in `docs/design-system.md`.
4. **Value format**: use OKLCH, HSL, or hex — whichever the project's design bundle uses. Document the format choice in `docs/design-system.md` under "Color Format."
5. **Spacing and type scale**: these do not live as CSS custom properties in most Tailwind projects — they live in `tailwind.config.ts`. See `references/tailwind-config-template.md`.
