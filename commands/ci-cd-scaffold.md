# /ci-cd-scaffold

Load and follow the [`sp-ci-cd-scaffold`](../skills/sp-ci-cd-scaffold/SKILL.md) skill.

The skill bootstraps the CI/CD and E2E baseline that every later feature slice depends on. It is the canonical Stage 2 of any phased plan generated by `prd-to-phased-plans`:

1. Creates a dedicated `chore/ci-cd-scaffold` branch.
2. Installs and configures Playwright with `@feature` and `@regression-core` tag suites.
3. Writes GitHub Actions workflows: `ci.yml`, `e2e.yml`, and `e2e-coverage.yml`.
4. Configures Husky pre-push hooks and a PR template.
5. Provides a branch-protection setup script.
6. Opens a PR to `main` and verifies all checks pass before returning.

## Preconditions

- Working tree is clean on `main`.
- `gh` CLI is installed and authenticated.
- Stage 1 (design system gate) is complete.

## When to use this command

Use `/ci-cd-scaffold` to run Stage 2 manually. The orchestrator (`/orchestrator`) invokes this automatically. This skill writes infrastructure, not product code — `sp-feature-delivery` begins only after this stage merges.
